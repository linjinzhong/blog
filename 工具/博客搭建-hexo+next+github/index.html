记录下搭建个人博客流程，使用 hexo + next + github + 个人域名。

前期准备

  1. 全局替换如下信息
  2. 腾讯云 cvm-centos:7
    1. 购买初始化
    2. 替换腾讯云源
    3. 磁盘映射-samba
      1. 安装
      2. 选择共享文件夹
      3. 编辑samba配置文件
      4. 创建samba用户名和密码
      5. 设置为开启启动
      6. 重启sambda
    4. cvm -> 安全组 -> 自定义模板 -> 按模板格式放开端口
  3. 腾讯云域名
    1. 注册
    2. 添加A记录
    3. 添加CNAME
  4. github
    1. 注册和登录
    2. 新建仓库blog
    3. 创建分支
    4. 配置git
    5. 新建ssh keys
    6. 配置host链接git更快

github部署

  1. 编辑_config.yml
  2. 部署推送git的master分支
  3. github配置
  4. 配置自定义域名
    1. 编辑_config.yml
    2. github配置腾讯云域名
    3. 重新部署推送
    4. 问题
      1. 发现master分支下的CNAME文件不见了
      2. 同时Pages -> Custom domain下自定义的域名也被重置为一开始默认的
    5. 解决方法
      1. 将生成的CNAME文件放在/data/service/blog/source下
      2. 再次部署推送发现本地.deploy_git目录下就有CNAME文件
      3. 同时master分支上的内容就是该.deploy_git目录下内容
    6. 最终可以通过个人域名访问hexo博客
  5. 源代码保存在默认的source分支
    1. 目的是可以再任意地方重新部署写文章
    2. 关联远程的默认source分支
  6. 迁移到其他CVM
